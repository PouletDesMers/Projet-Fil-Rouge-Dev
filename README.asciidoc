### Architecture Logique (API Go)

L'API suit un modèle de conception en couches :

`Routes` → `Middlewares` → `Handlers` → `Services` → `Repositories` → `PostgreSQL`

Les middlewares s'exécutent dans cet ordre :
1. SecurityHeaders (CSP, CORS, XSS)
2. MaxBodySize (limite à 10MB)
3. CORS
4. RateLimitAPI (100 requêtes/min par session)
5. RequestLogger (enregistrement en DB + fichier)

### Architecture des middlewares Node.js

Pour les routes admin, le proxy Node.js applique :
1. `checkAdminAuth` → vérifie l'admin (avec cache 30s)
2. `strictLimiter` → 120 requêtes/min sur `/admin/api`
3. `apiLimiter` → 100 requêtes/min sur `/api`

## 🚀 Démarrage Rapide

### Prérequis

* **Docker** & **Docker Compose** installés
* Aucune installation locale de Go ou Node.js n'est requise

### Installation

1. **Cloner le dépôt**
[source,bash]
----
git clone https://github.com/PouletDesMers/Projet-Fil-Rouge-Dev.git
cd Projet-Fil-Rouge-Dev
----

2. **Configuration**
Créer un fichier `.env` à la racine :
[source,env]
----
# Stripe (paiement)
STRIPE_SECRET_KEY=sk_test_...
STRIPE_PUBLISHABLE_KEY=pk_test_...

# API (optionnel - secret pour JWT)
API_SECRET=your-secret-key

# Email (Resend)
RESEND_API_KEY=re_...

# Backup
BACKUP_INTERVAL_MINUTES=60
----

3. **Lancement**
[source,bash]
----
docker-compose up --build
----

## 💾 Système de Sauvegardes

Les sauvegardes de la base de données sont gérées via `pg_dump` :

### Types de sauvegarde

* **full** (`full_*.sql`) — Toutes les tables
* **data** (`data_*.sql`) — Toutes les tables sauf `api_logs`
* **logs** (`logs_*.sql`) — Uniquement la table `api_logs`
* **auto_full** (`auto_full_*.sql`) — Sauvegarde automatique complète

### Sauvegarde manuelle

Via le panel admin → Sauvegardes → "Lancer la sauvegarde"

### Sauvegarde automatique (planification)

Planification avec intervalle en minutes via :
* Interface admin : sélecteur `min/h`
* Variable d'environnement : `BACKUP_INTERVAL_MINUTES=60`

Les règles sont persistées dans `/backups/schedules.json`.

### Restauration

Depuis l'interface admin : téléchargement ou restauration directe via `psql`.

### Volumes Docker

[source,yaml]
----
volumes:
  postgres_data:   # Données PostgreSQL (persistant)
  uploads_data:    # Images uploadées
  backups_data:    # Sauvegardes (.sql files)
  api_logs:        # Fichier de logs API (JSONL)
----

## 📝 Système de Logs

Les logs API sont écrits dans **3 stockages simultanément** :

[cols="1,2,2", options="header"]
|===
| Stockage | Persistance | Accès
| **RAM** (500 entrées) | Perdue au redémarrage | Fallback si DB indisponible
| **PostgreSQL** (table `api_logs`) | 90 jours (purge auto) | Interface logs + SQL
| **Fichier** (`/var/log/api/api.log`) | Illimitée (JSONL) | `docker exec` + `tail`
|==|

### Fonctionnalités de l'interface logs

* Cache local incrémental : 5000 logs initiaux, 50 par refresh
* Auto-refresh (60s), export CSV complet
* Filtres : niveau, méthode, statut HTTP, date, texte
* Stats en temps réel par niveau

## ⚡ Cache Frontend

Le panel admin intègre un cache API côté client automatique :

* **`AdminApiCache`** — Cache mémoire avec TTL 30s
* **Cache automatique** : toutes les `GET /admin/api/*` sont mises en cache
* **Invalidation automatique** : `POST/PUT/DELETE` → vide le cache concerné
* **Exclusions** : endpoints logs et backup (données temps réel)

## 🔐 Sécurité

### Rate Limiting

**Node.js (express-rate-limit) :**
[cols="1,2,2", options="header"]
|===
| Limiteur | Limite | Routes
| `loginLimiter` | 5 requêtes/15min | `/auth/login`
| `apiLimiter` | 100 requêtes/min | `/api/*`
| `strictLimiter` | 120 requêtes/min | `/admin/api/*`
|===

**Go API :**
[cols="1,2,2", options="header"]
|===
| Limiteur | Limite | Routes
| `APILimiter` | 100/min par session | Toutes les routes
| `AdminLimiter` | 600/min par session | Routes admin
| `LoginLimiter` | 5/min par IP | `/api/login`
|===

### Circuit Breaker (Frontend)

En cas de réponse 429, le frontend bloque toutes les requêtes API pendant la durée du `Retry-After` avec un bandeau rouge et compte à rebours.

## 📦 Commandes Docker Utiles

[source,bash]
----
# Construire et démarrer
docker-compose up --build -d

# Voir les logs d'un service
docker-compose logs -f api
docker-compose logs -f web

# Logs fichier API
docker-compose exec api tail -f /var/log/api/api.log

# Lister les sauvegardes
docker-compose exec api ls -lh /backups/pgdump/

# Console PostgreSQL
docker-compose exec db psql -U postgres -d mydb

# Rebuild un service
docker-compose build api && docker-compose up -d
docker-compose build web && docker-compose up -d

# Arrêt sans perte de données
docker-compose down

# Arrêt + suppression volumes (⚠️ perte données)
docker-compose down -v
----

## 🧪 Tests & CI

### Exécution locale

[source,bash]
----
# Tests Go
cd api && go test ./... -v

# Lint Go
cd api && go vet ./...

# Tests Node.js
cd web && npm test
----

### Pipeline CI

Le workflow GitHub Actions valide chaque modification :
1. **Linter :** `go vet`
2. **Build :** Compilation des binaires
3. **Test :** Suites de tests automatisées
4. **Docker :** Construction des images

### Pentest intégré

Endpoint `/admin/api/security/pentest` vérifie :
- Headers de sécurité (CSP, HSTS, X-Frame-Options...)
- Rate limiting (login, API)
- Protection CSRF
- Blocage accès anonyme
- Spécification Swagger
