CYNA est une plateforme SaaS de cybersécurité permettant aux entreprises de souscrire à des solutions de protection (EDR, XDR, SOC). Le projet est architecturé autour de trois services conteneurisés communicants.

## 🏗 Architecture du Projet

### Structure des répertoires

[source,text]
----
Projet-Fil-Rouge-Dev/
├── api/                # API REST Go (Logique métier)
├── web/                # Node.js (Proxy & Interface Admin)
├── db/                 # Scripts d'initialisation SQL
├── .github/            # Pipelines CI/CD
└── docker-compose.yml  # Orchestration des services
----

### Architecture Logique (API Go)

L'API suit un modèle de conception en couches pour une séparation stricte des préoccupations :

`Routes` → `Handlers` → `Services` → `Repositories` → `PostgreSQL`

NOTE: Un pipeline de middlewares intercepte les requêtes pour l'authentification, le rate limiting et la gestion des erreurs.

## 🚀 Démarrage Rapide

### Prérequis

* **Docker** & **Docker Compose** installés.
* Aucune installation locale de Go ou Node.js n'est requise.

### Installation

1. **Cloner le dépôt**
[source,bash]
----
git clone [https://github.com/PouletDesMers/Projet-Fil-Rouge-Dev.git](https://github.com/PouletDesMers/Projet-Fil-Rouge-Dev.git)
cd Projet-Fil-Rouge-Dev
----

2. **Configuration**
Créer un fichier `.env` à la racine :
[source,env]
----
STRIPE_SECRET_KEY=sk_test_...
STRIPE_PUBLISHABLE_KEY=pk_test_...
----

3. **Lancement**
[source,bash]
----
docker-compose up --build
----

## 🔐 Sécurité

La plateforme implémente les standards de sécurité suivants :

* **Auth :** Tokens de session opaques via Header `Authorization`.
* **2FA :** Authentification à deux facteurs via TOTP.
* **Protection :** Rate limiting par adresse IP.
* **Cryptographie :** Mots de passe hashés (Bcrypt) et sauvegardes chiffrées (AES-256 via Restic).

## 🧪 Tests & CI

### Exécution locale

* **Go :** `cd api && go test ./... -v` (13 tests unitaires couvrant `middleware` et `errors`).
* **Node.js :** `cd web && npm test`.

### Pipeline CI
Le workflow GitHub Actions valide chaque modification :
1.  **Linter :** `go vet`.
2.  **Build :** Compilation des binaires.
3.  **Test :** Exécution des suites de tests automatisées.
